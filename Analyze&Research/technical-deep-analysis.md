## 1. CSI 기반 인식 기술 원리

### 1.1 WiFi Channel State Information (CSI) 개요

WiFi CSI는 802.11 OFDM 물리 계층에서 각 서브캐리어(subcarrier)별로 측정된 복소 채널 응답으로, 단순 수신 신호 강도(RSSI)와 달리 **진폭(amplitude)과 위상(phase) 정보를 동시에 포함**합니다.

```
H[k] = A[k] · e^(j·φ[k])
```

- `H[k]`: k번째 서브캐리어의 채널 응답
- `A[k]`: 진폭 (신호 감쇠 반영)
- `φ[k]`: 위상 (경로 지연 반영)
- `k`: 서브캐리어 인덱스 (ESP32-S3 기준 64개, 20MHz 채널폭)

**RSSI 대비 CSI의 정밀도:** 단일 스칼라값인 RSSI에 비해 ESP32-S3는 64개 서브캐리어 × I/Q 두 채널 = **128차원 벡터**를 제공, 정보량 기준 약 **64배 향상**.

### 1.2 인체 반사 메커니즘

인체가 WiFi 신호 경로 내에 있을 때, 신체 표면과 내부 조직에서 다중 반사가 발생합니다.

**주요 변위 크기:**

| 동작 | 변위 크기 | 감지 주파수 대역 |
|------|----------|----------------|
| 흉부 호흡 운동 (2.4GHz) | 10–20 mm | 0.1–0.5 Hz |
| 흉부 호흡 운동 (5GHz) | 5–10 mm | 0.1–0.5 Hz |
| 심장박동 표면 변위 | 0.1–0.5 mm | 0.8–2.0 Hz |
| 보행 (전신 이동) | 수십 cm | 1–2 Hz |
| 손짓/제스처 | 수 cm | 3–10 Hz |

**Fresnel Zone 모델 — 핵심 수식:**

```
Fresnel Zone 반경:  F_n = √(n · λ · d₁ · d₂ / (d₁ + d₂))
위상 변화량:        ΔΦ = 2π · 2Δd / λ
```

- `λ`: 파장 (2.4GHz → 12.5cm, 5GHz → 6cm)
- `d₁`, `d₂`: TX–물체, 물체–RX 거리
- `Δd`: 신체 변위량

5GHz에서 5mm 흉부 변위 시 위상 변화: `ΔΦ = 2π × 0.01 / 0.06 ≈ 1.047 rad (약 60°)` — 충분히 감지 가능한 수준.

### 1.3 다중 경로(Multipath) 활용

직접파(LOS) 외에 벽, 가구, 바닥 반사파를 **신호 풍부화(multipath richness)**로 활용합니다. 실내 환경에서는 오히려 NLOS(비가시선) 반사가 인체 위치 추정 정보를 풍부하게 합니다.

- **단점:** 금속 장애물, 이동 가구에 의한 비인체 반사 혼입
- **대응:** Hampel 필터(이상값 제거) + Fresnel 기하 모델(인체 반사 특성 필터링)

### 1.4 채널 다이버시티(Channel Diversity)

ESP32-S3의 채널 호핑(channel hopping) 기능을 통해 2.4GHz 채널 1/6/11을 순차 탐색합니다.

- **드웰 타임:** 채널당 50ms
- **목적:** 특정 채널의 간섭·페이딩을 다른 채널로 보완
- **다중 밴드 융합(Multi-Band Fusion):** 3개 채널 CSI를 가중 평균으로 결합 → 단일 채널 대비 신뢰도 향상

## 2. 신호처리 파이프라인 상세

### 2.1 전체 파이프라인 개요

```
ESP32 UDP 스트림 (148 bytes/frame @ ADR-018 포맷)
  ↓
HardwareNormalizer → 정규화된 56-서브캐리어 프레임
  ↓
SOTA 전처리 6종 (Hampel, Fresnel, BVP, Spectrogram 등)
  ↓
RuvSense 6단계 파이프라인 → 20Hz 출력
  ↓
신경망 추론 → 17 COCO 키포인트 + DensePose
```

### 2.2 SOTA 알고리즘 6종 (ADR-014)

#### 알고리즘 1: Conjugate Multiplication (CSI Ratio Model)

**목적:** CFO(반송파 주파수 오프셋), SFO(샘플링 주파수 오프셋) 제거

```
CSI_ratio[k] = H_1[k] × conj(H_2[k])
```

- `H_1`: 기준 안테나/서브캐리어 채널
- `H_2`: 비교 채널
- **참고 논문:** SpotFi (SIGCOMM 2015), IndoTrack (MobiCom 2017)
- **구현 모듈:** `csi_ratio.rs`

#### 알고리즘 2: Hampel Filter

**목적:** 시계열 CSI 이상값(burst error, 간섭 스파이크) 제거

```
중위수 절대편차: MAD = median(|x_i - median(x)|)
표준편차 추정:   σ_est = 1.4826 × MAD
이상값 판정:     |x_i - median| > k × σ_est
```

- **오염 내성:** 50%까지 견딤 (Z-score는 이상값에 의해 쉽게 오염)
- **참고 논문:** WiGest (2015), WiDance (2017)
- **구현 모듈:** `hampel.rs`

#### 알고리즘 3: Fresnel Zone Breathing Model

**목적:** 호흡에 의한 흉부 변위 → CSI 위상 변화 모델링

```
F_n = √(n · λ · d₁ · d₂ / (d₁ + d₂))
ΔΦ = 2π · 2Δd / λ
```

- **참고 논문:** FarSense (MobiCom 2019), Wi-Sleep (UbiComp 2021)
- **구현 모듈:** `fresnel.rs`

#### 알고리즘 4: CSI 스펙트로그램 (STFT)

**목적:** 시간-주파수 2D 행렬 생성 → CNN 입력

| 주파수 대역 | 대응 동작 |
|------------|----------|
| 0.1–0.5 Hz | 호흡 |
| 0.8–2.0 Hz | 심박 |
| 1–2 Hz | 보행 |
| 3–10 Hz | 제스처 |

- **구현 모듈:** `spectrogram.rs`

#### 알고리즘 5: Subcarrier Sensitivity Selection

**목적:** 동작 감지에 민감한 서브캐리어 선별 → SNR 향상

```
sensitivity[k] = var_motion(amp[k]) / (var_static(amp[k]) + ε)
```

- **SNR 향상:** 6–10 dB (상위 K개 서브캐리어 선택 시)
- **구현 모듈:** `subcarrier_selection.rs`

#### 알고리즘 6: Body Velocity Profile (BVP)

**목적:** 인체 도플러 속도 프로파일 추출

```
v = f_doppler × λ / 2
```

- **표현:** 속도 × 시간 2D 행렬
- **장점:** 환경(방 레이아웃)에 독립적인 도메인 불변 표현
- **참고 논문:** Widar 3.0 (MobiSys 2019)
- **구현 모듈:** `bvp.rs`

### 2.3 RuvSense 6단계 파이프라인 (ADR-029)

#### Stage 1: Multi-Band Fusion (`multiband.rs`)

| 파라미터 | 값 |
|---------|---|
| 융합 윈도우 | 200ms |
| 대상 채널 | 1, 6, 11 (2.4GHz) |
| 드웰 타임 | 50ms/채널 |
| 최소 코히런스 | 0.30 |

채널 간 교차 코히런스 점수(0.0–1.0)를 산출하여 신뢰도 낮은 채널 배제.

#### Stage 2: Phase Alignment (`phase_align.rs`)

- **알고리즘:** `ruvector-solver::NeumannSolver` (O(√n) 희소 선형 해법)
- **목적:** 노드 간 LO(Local Oscillator) 위상 오프셋 보정
- **수렴 조건:** tolerance = 1e-5, 최대 반복 = 500

#### Stage 3: Multistatic Fusion (`multistatic.rs`)

| 파라미터 | 값 |
|---------|---|
| 최소 노드 수 | 2개 (멀티스태틱 모드) |
| 가드 인터벌 | 5ms (타임스탬프 허용 오차) |
| 어텐션 온도 | 1.0 |
| 인원 분리 | 활성화 |

- **크로스 노드 가중치:** `ruvector-attn-mincut` 물리 기반 어텐션
- **출력:** `FusedSensingFrame` (노드 간 코히런스 점수 포함)

#### Stage 4: Coherence Scoring (`coherence.rs`)

```
score = Σ(w_i × exp(-0.5 × z_i²)) / Σ(w_i)
z_i = |current_i - reference_i| / √(variance_i)
```

- **기준 템플릿:** EMA(지수이동평균), decay = 0.95
- **교정 요구량:** 최소 10분 @ 20Hz = **12,000 프레임** (Welford 온라인 통계)

#### Stage 5: Coherence Gating (`coherence_gate.rs`)

| 결정 | 임계값 | 처리 방식 | 노이즈 배율 |
|-----|--------|---------|-----------|
| **Accept** | > 0.85 | 완전한 칼만 측정 업데이트 | 1.0× |
| **PredictOnly** | 0.50–0.85 | 칼만 예측만 수행 | 3.0× |
| **Reject** | < 0.50 | 측정값 폐기 | — |
| **Recalibrate** | 10초 연속 저신뢰도 | SONA/AETHER 재교정 트리거 | — |

Recalibrate 조건: **200 stale frames @ 20Hz = 10초** 연속 저신뢰도.

#### Stage 6: Pose Tracking (`pose_tracker.rs`)

- **키포인트:** 17개 COCO 스켈레톤
- **상태 벡터:** 키포인트당 6D = [x, y, z, vx, vy, vz]
- **공분산:** 6×6 → 21개 원소 상삼각 행렬(f32) 압축 저장
- **운동 모델:** 등속도(constant-velocity) + 프로세스 노이즈

**트랙 생애주기:**
```
Tentative(신규 후보) → Active(안정 추적) → Lost(일시 소실) → Terminated
```

**재식별(Re-ID) 설계:**

| 파라미터 | 값 |
|---------|---|
| AETHER 임베딩 차원 | 128-dim |
| 재식별 윈도우 | 5초 (100 프레임 @ 20Hz) |
| 비용 함수 | 60% Mahalanobis + 40% AETHER 코사인 유사도 |
| 트랙 생성 조건 | 연속 2프레임 감지 (단발 노이즈 제거) |
| 트랙 소실 판정 | 연속 5프레임 미감지 |

**다인원 분리:** `ruvector-mincut::DynamicMinCut` — 크로스링크 상관관계 그래프 구성.

## 3. 신경망 아키텍처 상세

### 3.1 전체 모델 구조

WiFi CSI → 시각적 특징 공간으로의 **모달리티 변환(Modality Translation)** 후, 컴퓨터 비전 기반 DensePose 파이프라인을 적용하는 2단계 구조입니다.

```
CSI 입력 텐서 [32, 9, 56, 100]
  ↓
ModalityTranslator (CSI → 256-dim 시각 특징)
  ↓
ResNet18 Backbone
  ↓
Feature Pyramid Network (FPN)
  ↓
Region Proposal Network (RPN)
  ↓
ROI Align
  ├── KeypointHead → 17 COCO 관절
  └── DensePoseHead → 24 신체 부위 + UV 좌표
```

### 3.2 입력 텐서 사양

| 차원 | 값 | 설명 |
|-----|---|-----|
| Batch | 32 | 미니배치 크기 |
| Antenna | 9 | 안테나 페어 (3×3 MIMO 또는 가상 확장) |
| Subcarrier | 56 | 정규화된 서브캐리어 수 |
| Temporal | 100 | 시간 윈도우 프레임 수 (5초 @ 20Hz) |

**이중 스트림 입력:**
- Amplitude 브랜치: CSI 크기 정보
- Phase 브랜치: CSI 위상 정보 (SpotFi 보정 후)

### 3.3 ModalityTranslator

CSI 신호를 256차원 시각 특징 벡터로 매핑하는 핵심 변환 모듈.

```
FC층 구성: [256 → 512 → 256]
출력: 256-dim visual feature space
최종 합성 해상도: 720 × 1280 × 3 (가상 이미지 공간)
```

**학습 방식:** 교사-학생(Teacher-Student) Knowledge Distillation
- 교사: 실제 RGB 이미지 + 기존 DensePose 모델
- 학생: CSI 입력 ModalityTranslator
- 손실 함수: MSE (특징 공간) + Cross-entropy (포즈 라벨)

### 3.4 KeypointHead

| 파라미터 | 값 |
|---------|---|
| 입력 채널 | 256 |
| 히든 채널 | [256, 128, 64] |
| 출력 | 17 COCO 관절 히트맵 |
| 공간 어텐션 | `ruvector-attention` (장거리 의존성) |

**17 COCO 관절:**
코, 왼눈/오른눈, 왼귀/오른귀, 왼어깨/오른어깨, 왼팔꿈치/오른팔꿈치, 왼손목/오른손목, 왼엉덩이/오른엉덩이, 왼무릎/오른무릎, 왼발목/오른발목

### 3.5 DensePoseHead

| 파라미터 | 값 |
|---------|---|
| 입력 채널 | 256 |
| 신체 부위 수 | 24 (NUM_BODY_PARTS) |
| UV 좌표 수 | 2 (NUM_UV_COORDINATES) |
| 출력 형식 | 부위 분류 맵 + UV 표면 좌표 |

DensePose는 인체 표면을 24개 의미적 영역으로 분할하고 각 픽셀에 3D 표면 좌표(U, V)를 부여합니다. CSI 기반으로 이를 추론함으로써 **단순 스켈레톤을 넘어선 연속 표면 재현**이 가능합니다.

### 3.6 RuVector 통합 성능 이점 (ADR-016)

| RuVector 크레이트 | 사용처 | 성능 이점 |
|-----------------|-------|---------|
| `ruvector-mincut` | 인원 분리, 트랙 할당 | O(n^1.5 log n) vs O(n³) |
| `ruvector-attn-mincut` | 안테나 가중치 | 물리 기반 페어 선택 |
| `ruvector-temporal-tensor` | CSI 버퍼 압축 | 50–75% 메모리 절감 |
| `ruvector-solver` | 위상 정렬, 서브캐리어 보간 | O(√n) vs O(n) |
| `ruvector-attention` | 공간 디코더 | 장거리 의존성 포착 |

### 3.7 학습 데이터셋 (ADR-015)

| 데이터셋 | 규모 | 특징 |
|---------|-----|-----|
| MM-Fi | 40명, 14개 동작, 5개 환경 | WiFi CSI + mmWave + RGB 동시 수집 |
| Wi-Pose | 10개 환경, 다양한 거리 | WiFi 전용 포즈 라벨 |

**도메인 일반화 (ADR-027, MERIDIAN):** 훈련 환경과 다른 실내 공간에서의 성능 저하 완화를 위한 적대적 도메인 적응 적용.

**AETHER 임베딩 (ADR-024):** 대조 학습(Contrastive Learning) 기반 128-dim 신체 형태 식별 벡터 — 다중 인원 재식별에 활용.

| ESP32-S3 SuperMini (4MB) | 64 | SISO 1×1 | 20/40 MHz | `Esp32S3` | ~$6 |

**미지원:** ESP32 오리지널, ESP32-C3 — 단일 코어로 CSI DSP 파이프라인 실행 불가.

### 4.2 ESP32-S3 상세 사양

**메모리 구성:**

| 메모리 종류 | 용량 | 용도 |
|-----------|-----|-----|
| 온칩 SRAM | 320 KB | 펌웨어 실행, 스택 |
| PSRAM (옵션) | 8 MB | CSI 프레임 버퍼링 |
| Flash | 8–16 MB | 펌웨어 저장, OTA |

**WiFi 사양:**
- 802.11 a/b/g/n (2.4 GHz + 5 GHz)
- 안테나: SISO 또는 GPIO 전환 이중 RX
- CSI 측정: 64 서브캐리어 × I/Q = 128 복소수 값/프레임

**전력 소비:**
- 연속 CSI 수집 모드: **150–200 mA** (ADR-028 기준)
- 5V USB 전원 시 약 **0.75–1.0 W**

### 4.3 ADR-018 바이너리 프레임 포맷

ESP32가 UDP(포트 5005)로 전송하는 바이너리 프레임 구조:

```
오프셋  크기  필드              타입      값 범위
------  ----  -----             ----      -------
0       4     Magic             LE u32    0xC5110001
4       1     Node ID           u8        0–255
5       1     Antenna count     u8        1–4
6       2     Subcarrier count  LE u16    56/64/114/242
8       4     주파수 (MHz)       LE u32    2412–5825
12      4     시퀀스 번호        LE u32    단조증가
16      1     RSSI              i8        dBm
17      1     Noise floor       i8        dBm
18      2     Reserved          [u8;2]    0x00 0x00
20      N×2   I/Q 페이로드       [i8;2*n]  안테나×서브캐리어
```

**전형적 프레임 크기 (ESP32-S3):** 20바이트 헤더 + 64 서브캐리어 × 2바이트 = **148 바이트**

### 4.4 펌웨어 아키텍처 (ESP-IDF v5.4, C)

| `csi_collector.c` | 176 | CSI 콜백 처리 |

**TDMA 동기화:** 50ms 사이클, 최대 255 노드 (node_id: u8)

### 4.6 하드웨어 한계 분석

**1. SISO 제약 (가장 큰 한계)**
- ESP32-S3는 단일 안테나(SISO 1×1)로 공간 다이버시티 없음
- Intel 5300 (MIMO 3×3, 30 서브캐리어)과 달리 각도 분해능(AOA) 불가
- **대응:** 다중 노드 배치로 공간 다이버시티 확보 (ADR-029 멀티스태틱)

**2. 채널 호핑 지연**
- 3개 채널 순환 시 실질 단일 채널 갱신률: 약 6.7Hz
- 빠른 움직임(3Hz 이상 주파수 성분) 감지 한계
- **대응:** 단일 채널 고정 시 최대 ~100Hz 가능 (정밀 측정 모드)

**3. 교정 시간 요구**
- Welford 통계 수렴: 최소 **12,000 프레임 = 10분** 필요
- 공간 변경(가구 이동 등) 시 재교정 필요
- Recalibrate 트리거: 10초 연속 저신뢰도

**4. 금속 장애물 취약성**
- 금속 가구, 금속 벽 → 비인체 반사로 오감지
- Hampel 필터 + Adversarial 감지 모듈(`adversarial.rs`)로 부분 완화

**5. 멀티노드 동기화**
- 가드 인터벌: 5ms — 노드 간 타임스탬프 편차가 이를 초과하면 퓨전 실패
- NTP 또는 PTP 동기화 필요 (현재 펌웨어: NVS 저장 기반, ADR-028 갭)

### 5.2 Coherence Gate 정확도 구조

**게이팅에 따른 추정 품질:**

```
Accept  (≥0.85): 완전 칼만 업데이트 → 최고 정확도
PredictOnly (0.50–0.85): 예측만 → 트랙 유지, 정확도 점진 저하
Reject (<0.50): 측정 폐기 → 관성 예측만
Recalibrate: 10초 이상 → 기준 템플릿 재구축
```

PredictOnly 구간에서 노이즈 배율 3.0× 적용 → 공분산이 빠르게 팽창하여 시스템이 불확실성을 적절히 표현함.

### 5.4 포즈 추정 한계

**거리별 성능:**
- 최적 범위: **1–5 m** (Fresnel zone 내, 직접 반사 강도 충분)
- 5 m 초과: 정확도 저하 문서화 없음 — **미검증**
- 근거리 (<1 m): 다중 경로 혼재로 오히려 성능 저하 가능

**인원수별 성능:**
- 1인: 주요 개발·검증 시나리오
- 2인: `ruvector-mincut` 분리 지원
- 3인 이상: 크로스링크 상관 복잡도 급증 — **오탐률 미정량화** (ADR-028 갭)

**동작별 한계:**

| 동작 | 감지 품질 | 비고 |
|-----|---------|-----|
| 보행 | 우수 | 1–2 Hz 도플러 명확 |
| 제자리 서기 | 양호 | 호흡·미세 움직임 의존 |
| 빠른 손 동작 | 제한적 | 채널 호핑 시 6.7Hz 한계 |
| 완전 정지 | 취약 | 코히런스 저하 → PredictOnly |
| 누운 자세 | 제한 | 토르소-안테나 방향 의존 |

### 5.5 교정 요구사항 분석

